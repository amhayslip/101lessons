Adding Our Form

First we need an input (usually inputs go inside forms) so the user can input a new Todo. Add the following form tag to your HTML (don't copy/paste the "page-header" part becuase it's already in the code... it's below so you know where to paste the form code):

  <h1>Todo App</h1>
</div>

<form>
 <h3>Add a new post</h3>
 <input type="text" class="form-control todo-input" placeholder="Todo"></input>
</form>

The form-control class on the input is from Bootstrap, and just adds some extra style. We'll use the todo-input class later to identify the input with our jQuery.

Now let's add a button so the user can actually submit the new input they type in. Make your form HTML look like the following:

<form>

</form>

Again, btn and btn-primary are just Bootstrap classes that make our button prettier. The post-todo class will help us identify the button with jQuery in a moment. If you haven't already, open this up in the browser to see how it looks. It should look like this:

Displaying Our Todo's

Finally, now we need a place to actually display our todo's. Add the following HTML to your page between your page-header and form:

<ul class="todo-list">
  <li>Do some stuff!</li>
</ul>

Once again, with your partner, see if you can figure out the necessary jQuery to actually append the value of the input to the "todo-list" on the page before peeking at the answer. Don't spend too much time on this, as there's a catch.

$('.post-todo').on('click', function (e) {
  e.preventDefault(); 
  var todo = $('.todo-input').val();
  $('.todo-list').append('<li>' + todo + '</li>');
});

This last one takes some explanation, starting with the e.preventDefault() . This is more of a quirk than anything else. Some DOM nodes (HTML elements) have default events attached to them. The button element is one of those. By default, it wants to submit a form. By passing in the e argmument (stands for event) and calling preventDefault() on it, we can prevent it from it's default behavior and make it do what we want it to do.

Lastly, notice our append.() method. It accepts a string as an argument. We can give it an HTML string with our variable interpolated and it will append a new HTML li element to our ul element.
