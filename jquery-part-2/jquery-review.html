Event Listeners

Also from last time, Event Listens are methods that listen for events, such as clicks! Below is an example of a button listening for click events using the .on method:

$('button').on('click', function () {
  //empty function body
});

Let's break it down a bit. First we're selecting the button element like this:

$('button')

If you remember back to last time, selecting it with jQuery means now it has a bunch of methods available for it to use (a method is an object property that's value is a function). One of those methods is the on method. You can imagine it looking like this:

var $('button') = {
  on: function () {
    //does some stuff
  }
}

In the example above, how would we access the on property of the $('button') object?

$('button').on

Cool. Now how would we call (invoke) the on function?

$('button').on()

Awesome. Because we're smart and we read the jQuery documentation on their website, we know that the on method takes two arguments: an event, and a function.

$('button').on(event, function)

We can find a list of events on the jQuery website, but we know that a common one is the click event. We pass it in as a string:

$('button').on('click', function)

Now for the function. Usually, when we see functions, we see them defined with a variable and they have a name. Like this:

var greet = function () {}

Usually we asign functions to variables (like the one above) so that we have a way to reference it later and "call it" (invoke it), like this:

greet();

However, with the function that we're passing to .on() as it's second argument, we don't need to give it a variable. Our .on() method know's how to reference it becuase it has it as an argument.

$('button').on('click', function () {});

We call this an anonymous function, because it has no name. Also, it's an example of a callback function because we're not calling it directly, but we're calling a function that call's it (the .on function calls it, not the $('button') object).

The keyword this

In the past, we've lightly covered the keyword this in javascript, but let's review. The keyword this is always used inside of a function and always refers to whatever called the function. For example:

var car = {
  position: 0,
  go: function () {
    this.position += 1
  }
}

car.go();

Above we have an object called car with two properites: position and go. The value of go is a function that changes the car's position. Inside that function, this is referring to the car object. Copy and paste the code into your Chrome Developer Console if you need to see it in action to understand what this is doing.

Let's now look at an example using jQuery 

$('button').on('click', function () {
  $(this).css('color', 'red');
});

Above, we wrapped this inside a jQuery selector to make it $(this). Since the button element called the function, $(this) refers to the button itself. Therefore, this function turns the button red.
