Blog, Todo, Comments, whatever... CRUD.

CRUD isn't just slang for feces. It stands for "Create", "Read", "Update" and "Delete". You can read more about the concept here in detail here, but these are the basics operations of most of your favorite web applications. Think (again) about Facebook and posts... users can create, read, edit and delete posts. Ultimately we want the ability to build applications that can do all of these things, and persist this data in a database. Today we started this journey by building a page where content can be dynamically Created. Yes, it's epic.

Movie Buff

Using what you learned today about jQuery, create a webpage called "Movie Buff" from scratch that displays users' favorite movies. Enable them to add movies using an input field and submit button. Style it any way you like and feel free to use bootstrap (or not).

Movie Buff Extension 1

Extensions are optional, so don't stress if you don't get here.

In additonal to an input that allows a user to enter the name of their favorite movie, also enable them to add a link to the trailer of the movie from youtube.

Movie Buff Extension 2

Create a Movie constructor function and create a new instance of the Movie using the user's inputted "movie title" and "trailer link". Append the new movie to the page using the object instance.

Movie Buff Extension 3

A good practice in programming is to make sure that our user interface reflects our data. Each time a new movie object is created, add it the a movies array. Then, create a renderMovie function that get's called when the movie object is created. The renderMovie function should first clear the movies div and then loop through the array and append each movie to the page.

Movie Buff Extension 4

Add a like button to each movie and keep track of the number of likes each movie has in a counter on the page.
